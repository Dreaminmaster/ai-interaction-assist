# VIDEO GAP ANALYSIS

## 当前版本已经做到的

当前 `local-app/` 已经具备这些基础能力：
- 一键本地启动
- 自动打开浏览器
- 主题创建与切换
- 本轮学习开始
- 回答提交
- 本轮沉淀更新
- 主题档案同步
- 本地文件持久化
- 导出备份 / 导入恢复
- 中文 / English 切换
- provider 切换

这些能力已经让它成为“可运行的本地交互式学习工具”，但它和视频里那种“交互式学习系统”相比，仍然有明显差距。

---

## 和视频风格相比，当前差距在哪里

### 1. 当前页面更像“工具页”，还不够像“学习工作台”
视频里的感觉通常是：
- 打开就能看到当前学习地图
- 当前任务、下一步、复习入口很明显
- 页面本身有连续学习感

当前版本更像：
- 左边配置
- 中间摘要和表单
- 右边状态

也就是说，功能有了，但“学习工作台感”还不够强。

### 2. 当前交互还是“单轮提交”，还不够像持续引导
视频里的感觉通常是：
- 系统一步一步引导
- 当前问题和下一步问题是连着的
- 不是简单输入一段文本然后看结果

当前版本还是：
- 开始本轮学习
- 回答一段话
- 更新总结

所以连续引导感还不够。

### 3. 当前知识结构展示不够强
视频里的风格通常会更强调：
- 学习路径
- 分层知识点
- 已完成 / 未完成
- 当前卡住点

当前版本虽有 topic、session、weakPoints，但：
- 没有真正的学习地图
- 没有节点式知识结构
- 没有明确的阶段推进视图

### 4. 当前“本轮沉淀”偏结果页，缺少过程感
现在的页面有：
- 本轮总结
- 已掌握
- 薄弱点
- 下次入口

这更像结果区。
视频里的交互系统通常会更强调：
- 当前正在做什么
- 下一步马上该做什么
- 为什么系统现在问这个

也就是更强的“过程引导”。

### 5. 当前系统还没有真正的学习地图生成
现在只有：
- 首轮诊断问题
- 回答分析
- 下次入口

但还没有：
- 自动拆分主题结构
- 分层学习路线
- 子模块展开
- 节点推进

这会让它和视频里的“系统感”还有差距。

---

## 最值得补的下一批能力

### A. 学习地图区
建议新增一个核心区块：
- 当前主题地图
- 基础层
- 理解层
- 应用层
- 表达层
- 已完成 / 进行中 / 待开始

这样页面一打开就不是“表单工具”，而是“学习系统”。

### B. 当前任务卡
建议把“本轮学习”改成更强的任务区：
- 当前任务是什么
- 为什么先做这个
- 完成标准是什么
- 下一步是什么

### C. 连续引导卡片
不要只放一个大输入框，可以改成：
- 第 1 问
- 你的回答
- 系统追问
- 小结
- 下一问

这样更接近视频里的交互式学习节奏。

### D. 学习进度条 / 节点状态
建议新增：
- 当前进度
- 已掌握节点数
- 当前薄弱节点数
- 最近推进的节点

### E. 系统主动生成下一层学习内容
现在只是“分析后更新总结”，还不够。
后续应让系统根据结果自动生成：
- 下一轮问题
- 下一层讲解
- 当前知识点解释
- 例子 / 类比 / 练习

---

## 当前结论

当前版本已经从“想法”推进到“可运行原型”，但如果要更接近视频里的那种交互式学习系统，接下来最重要的不是继续补零碎功能，而是：

1. 把页面从工具页改成学习工作台
2. 把交互从单轮提交改成连续引导
3. 把结构从摘要结果改成学习地图
4. 把系统输出从总结扩展到下一层学习内容生成

这四件事补起来之后，整体风格才会真正接近视频里的效果。
