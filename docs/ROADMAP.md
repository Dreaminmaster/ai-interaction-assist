# 开发路线图

## 1. 总体原则
第一版不要追求炫技，优先做通核心价值。

系统真正的核心不是页面效果，而是：
- 学习流程引擎
- 记忆引擎
- 主题档案引擎

路线图应遵循以下原则：
1. 先做最小可用版
2. 先保证学习连续性
3. 先保证本地可保存和可继续
4. 先做流程和记录，再做高级可视化

## 2. Phase 0：文档期
目标：把系统规则、产品范围和技术方案定清楚。

需要完成：
- README
- SYSTEM_PROTOCOL
- PRODUCT_SPEC
- ARCHITECTURE
- DATA_MODEL
- UI_FLOW
- ROADMAP

完成标准：
- 仓库中已有清晰的产品与架构文档
- 后续开发时无需反复重定方向

## 3. Phase 1：最小可用版 MVP
目标：做出一个本地一键打开、可以围绕主题连续学习的原型。

### 必做功能
- 本地应用壳子跑起来
- 首页可以新建主题
- 可以进入主题学习页
- 可以与 AI 连续对话
- 每轮结束后自动生成学习记录
- 本地保存主题档案
- 应用重开后可以继续

### 页面范围
- 首页
- 主题学习页
- 记录页基础版本

### 记忆范围
- 主题级别的基本记忆
- 上次摘要
- 当前薄弱点
- 下次入口

完成标准：
- 可以新建一个主题并持续学下去
- 可以关闭后再打开继续上次主题
- 本轮结束后能看到沉淀结果

## 4. Phase 2：主题档案与薄弱点增强
目标：让系统具备“越用越懂你”的基础能力。

### 增强内容
- 单独的薄弱点页
- 问题类型标注
- 薄弱点频率统计
- 主题档案增强
- 全局学习画像初版

### 新增页面
- 薄弱点页
- 主题档案增强视图

完成标准：
- 系统能识别常见错误类型
- 系统能持续追踪某个主题的长期薄弱点
- 不会每次都像新对话一样重新开始

## 5. Phase 3：复习系统
目标：把“学习完成后容易忘”这个问题纳入系统。

### 新增能力
- 复习页
- 错题复习
- 快速三问
- 薄弱点抽查
- 简单间隔复习机制

完成标准：
- 用户可以直接进入复习模式
- 系统能优先抽查未稳固内容
- 薄弱点状态可以从活跃变为已缓解

## 6. Phase 4：资料挂载与知识底座
目标：让系统不只是聊天，而是能围绕用户自己的资料学习。

### 新增能力
- 主题资料挂载
- Markdown 资料接入
- PDF 接入
- 图片接入
- 笔记关联

完成标准：
- 用户可把资料挂到主题下
- AI 在讲解和复习时可利用这些资料
- 主题学习不再只依赖即时对话

## 7. Phase 5：可视化与高级交互
目标：让产品更接近视频里的交互体验，甚至更好。

### 新增能力
- 学习地图可视化
- 知识树
- 主题进度展示
- 薄弱点热度展示
- 更丰富的卡片化学习页面
- 自动生成流程图和图表示意

完成标准：
- 用户对学习进度和结构有更直观的感受
- 产品体验明显超出普通聊天框

## 8. Phase 6：高级扩展
目标：让系统进一步产品化。

### 可选方向
- 语音输入与语音复习
- 多模型支持
- 可插拔 Provider 系统
- 输出资产管理
- 自动生成答辩稿 / 论文版 / 项目版文档
- 提醒系统
- 导出 Markdown / PDF 笔记

## 9. 开发顺序建议
按实际开发顺序，建议这样推进：
1. 文档全部就位
2. 跑通本地应用壳
3. 做首页和主题页
4. 跑通单轮学习流程
5. 跑通本轮沉淀
6. 加主题长期档案
7. 加薄弱点追踪
8. 加复习页
9. 加资料挂载
10. 最后做可视化

## 10. 版本优先级总结
### 当前最优先
- 本地一键打开
- 主题持续学习
- 本轮记录沉淀
- 主题档案保存

### 第二优先
- 薄弱点追踪
- 复习机制
- 学习画像

### 第三优先
- 资料挂载
- 图表和可视化
- 更炫的界面交互

## 11. 里程碑定义
### M1
文档完成，开发方向固定。

### M2
本地应用可启动，可新建主题，可开始学习。

### M3
可自动沉淀本轮记录，可继续上次学习。

### M4
可追踪薄弱点和问题类型。

### M5
可进入复习模式并完成抽查。

### M6
可挂载资料并形成更完整的主题知识底座。

### M7
可视化增强，整体体验接近甚至超过视频中的形态。

## 12. 最终目标
最终目标不是做一个“会聊天的壳子”，而是做一个真正长期可用的个人 AI 学习系统：
- 它知道你学到哪里
- 它知道你哪里没懂
- 它知道怎么继续教你
- 它能把你的学习过程积累成资产
- 它能在不同场景下把同一知识点转成不同表达

做到这些，这个系统就已经具有很强的独特价值。
